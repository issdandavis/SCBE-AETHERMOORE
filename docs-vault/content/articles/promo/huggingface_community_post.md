## Context

If you've looked at the SCBE-AETHERMOORE training data or models on this account, you may have noticed some unusual things: 6 constructed languages called "Sacred Tongues," references to neurotransmitter mappings, golden-ratio token weighting, and a 14-layer pipeline structure that seems too narratively coherent for a pure engineering project.

That's because it isn't a pure engineering project. It's also a novel.

## How the training data and the novel connect

The origin of SCBE-AETHERMOORE's training corpus is 12,596 paragraphs of AI-generated tabletop RPG logs from Everweave. Those logs were expanded into structured worldbuilding, then formalized into:

- **6 Sacred Tongues** (KO, AV, RU, CA, UM, DR) — each mapped to a neurotransmitter and a mathematical dimension, 256 tokens per language
- **A 14-layer security pipeline** using Poincare hyperbolic geometry for exponential cost scaling
- **Sacred Eggs** — controlled mutation gates for model genesis

This formalization became the SCBE-AETHERMOORE framework (open source on GitHub, provisional patent USPTO #63/961,403). The same formalization became the physics and magic system of the novel.

The SFT records in `issdandavis/scbe-aethermoore-training-data` are tagged with tongue dimensions and governance metadata that directly correspond to in-world concepts in the book. The PHDM embedding model (`issdandavis/phdm-21d-embedding`) operates in the same 21-dimensional space that characters navigate in the story.

## Why this matters for the ML community

This is an experiment in **narrative-seeded model training**. The hypothesis: training data generated from internally consistent fictional worlds with formal rule systems produces models with stronger coherence properties than data generated from unstructured sources.

The book is the human-readable version of that hypothesis. The training data is the machine-readable version. The code is the bridge.
