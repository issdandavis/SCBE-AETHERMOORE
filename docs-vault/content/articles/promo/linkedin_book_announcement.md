What if a governance framework became a novel?

Last year, I built an open-source AI security framework called SCBE-AETHERMOORE. It uses Poincare hyperbolic geometry to make adversarial behavior exponentially expensive. 14 layers of security. Post-quantum cryptography. Constructed languages as tokenizer seeds. I filed a provisional patent (USPTO #63/961,403).

Today, that framework is also a novel.

"The Six Tongues Protocol: Book One" is a 124,000-word isekai where a security engineer named Marcus Chen gets pulled into a world where magic operates through governance protocols. The 14-layer pipeline that governs AI agent behavior in my real codebase is the same system that governs spellcasting in the book. The 6 Sacred Tongues that function as tokenizer dimensions in the framework are the constructed languages characters speak in the story.

The fiction and the engineering are the same artifact.

How this happened: I played over 12,000 paragraphs of AI-powered tabletop RPG on Everweave. Those game logs became the seed corpus for a constructed language tokenizer. That tokenizer became a security framework. That framework became a novel. Each step generated the next.

I want to be transparent about where I'm writing from. I work at Wendy's. I live with my great-grandpa in Port Angeles, WA. I don't have a CS degree. I built this with consumer AI tools, open-source libraries, and an unreasonable amount of persistence.

The codebase is public on GitHub. The trained models are on HuggingFace. The book ties it all together as narrative.
