# Dense-bundle augmented coding/chemistry SFT

Generated by `scripts/encoding/augment_sft_with_dense_bundle.py` from
the source SFT files in `artifacts/kaggle_datasets/scbe-coding-agent-stage6-repair-v7/`.

## What was added per record

Each record gets a new top-level `dense_bundle` field with:

```json
{
  "dense_bundle": {
    "target": "user",
    "default_view": "hex",
    "route_lane": "binary_analysis",
    "intent_profile": {"neg": 0.006, "zero": 0.0, "pos": 0.994},
    "byte_length": 500,
    "density_ratio": 16.38,
    "views": {
      "hex": "...",
      "binary": "...",
      "base64": "...",
      "ternary": "..."
    },
    "intent": [1, 1, 1, ...]
  }
}
```

All four byte views round-trip back to the same payload bytes. The
intent overlay is parallel signal (one trit per byte: +1 / 0 / -1).

## How to regenerate

```bash
mkdir -p training-data/kaggle/dense_bundle_augmented
for f in artifacts/kaggle_datasets/scbe-coding-agent-stage6-repair-v7/*.sft.jsonl; do
  base=$(basename "$f" .sft.jsonl)
  PYTHONPATH=. python scripts/encoding/augment_sft_with_dense_bundle.py \
    "$f" "training-data/kaggle/dense_bundle_augmented/${base}.dense.jsonl"
done
```

## Stats

- Source: 29 SFT files / 3,370 records / 3.9 MB
- Output: 29 dense.jsonl files / 3,370 records / ~28 MB
- Density expansion: ~7x (binary view alone is 8x)

## Use cases

- Cross-view consistency training: train a model to produce the same
  answer regardless of whether the input is shown as text, hex, or
  base64 encoding
- Intent-layer training: route on the per-byte polarity signal
- Bus dispatch: `route_lane` tells the SCBE swarm router which lane
  this record's encoding form belongs to

See `notes/theory/2026-04-05-training-pair-taxonomy.md` row 10
("Bit/float/trit triple") for the underlying theory.
