What this lane is for

  • Run one useful conversation at a time. Draft, triage, rewrite, compare, and capture work without turning the phone into a theater diagram.
  • Use stronger models as training mirrors. Keep Polly as the main lane, add compare models when needed, then export thread, SFT, or feedback rows for your smaller models.
  • Keep host secrets off the emulator. The expected route is /v1/research/mobile-chat on the host. The host keeps the provider token; the phone just talks to the proxy.

Recommended prompts

  • Draft a customer reply about a delayed order and keep the tone calm.
  • Compare two versions of this answer and keep the stronger one for training.
  • Turn this thread into SFT-ready rows and point out weak assistant turns.

Proxy rule

Start the host API with SCBE_ALLOW_LOCAL_MOBILE_CHAT=1 and keep HF_TOKEN on the host. Use ?proxy=http://10.0.2.2:8000/v1/research/mobile-chat if you need to override the default.

Why this is different

The old arena treated the phone like a miniature command center. This version treats the phone like a real working surface: one main lane, optional compare models, and a straight path from conversation to training data.

Add ?secrets=1 only when you intentionally want the raw token or proxy-key fields visible for debugging.
