# Code of Conduct

SCBE-AETHERMOORE is a technical research and software project. Contributors are expected to keep collaboration focused, respectful, and useful.

## Expected Behavior

- Be direct, but professional.
- Discuss code, evidence, tests, designs, and risks rather than attacking people.
- Preserve user privacy, secrets, unpublished research, and operational security.
- Make security reports responsibly through the process in `SECURITY.md`.
- Assume good faith when possible, and ask for clarification when intent is unclear.

## Unacceptable Behavior

- Harassment, threats, intimidation, or targeted personal attacks.
- Publishing private information, credentials, tokens, or sensitive operational details.
- Repeated disruption of issues, pull requests, discussions, or review threads.
- Misrepresenting test results, security status, provenance, or authorship.
- Using project spaces to coordinate abuse, spam, malware, or unauthorized access.

## Scope

This applies to repository issues, pull requests, discussions, documentation, project communication channels, and any official project spaces.

## Reporting

For security vulnerabilities, follow `SECURITY.md`.

For conduct problems, open a private report through the repository owner or the project contact listed in the repository metadata. Include links, timestamps, screenshots, and a short description of what happened.

## Enforcement

Maintainers may remove content, close threads, block accounts, revoke access, or report abuse to platform providers when needed. Enforcement should be proportional, evidence-based, and aimed at keeping the project safe and productive.
