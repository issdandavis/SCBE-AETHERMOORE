# Delivery and Access

## What every buyer should receive

- product name
- bundle files
- matching package manual
- support contact
- access key or invite only if the product actually requires one

## Standard sequence

1. Receipt confirms the order.
2. Bundle arrives.
3. Buyer opens the matching package folder.
4. Buyer follows the package `README.md`.
5. Buyer completes the package success check.

## Practical rule

If a package is file-only, do not wait for a key.

If a package depends on a hosted route or private asset, the access information should be stated directly in delivery, not hidden in generic support text.

## Recovery

- Missing bundle or expired link:
  - `aethermoregames@pm.me`
  - subject: `SCBE Delivery Issue`
- Files arrived but are unclear or broken:
  - `aethermoregames@pm.me`
  - subject: `SCBE Package Support`
