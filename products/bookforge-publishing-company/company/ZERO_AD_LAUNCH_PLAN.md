# Zero-Ad Launch Plan

## Day 0: Proof

- Publish the proof page.
- Link the package page to the kit.
- Build the ZIP locally.
- Verify the wheel or PyPI install path.
- Pin the current KDP rule links in the quality gates.

## Day 1: Warm Launch

Post one clear offer:

> I built a $19 BookForge Publishing Kit for authors who want Markdown to KDP
> files without fighting trim sizes, spine math, EPUB, and DOCX export. It
> includes presets, starter manuscript, blurb template, and KDP upload checklist.

Use:

- GitHub README/package page.
- AetherMoore offers page.
- Ko-fi post.
- YouTube community or short demo.
- Existing author/writer contacts.

## Day 2: Demo

Record a two-minute screen capture:

1. Open starter manuscript.
2. Show profile.
3. Run `bookforge build`.
4. Show generated files.
5. Show KDP checklist.

## Day 3: Upsell

Offer the $99 Build Audit to anyone who says:

- "I have a manuscript but I am stuck."
- "KDP keeps rejecting my files."
- "I do not know what size my cover should be."

## Weekly Loop

- Add one new profile or template.
- Run the package build script.
- Refresh screenshots or receipts.
- Post one practical KDP/file-format note.
