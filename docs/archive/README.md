# docs/archive

Frozen working notes, draft narrative fragments, and historical cleanup logs.
Files here are kept for the git history they explain, not for current
guidance. None of them are wired into the documentation index, and they
should not be updated in place — if a topic in this directory becomes
load-bearing again, copy what's needed into `docs/` (or the relevant code
path) and leave the archived copy untouched.
