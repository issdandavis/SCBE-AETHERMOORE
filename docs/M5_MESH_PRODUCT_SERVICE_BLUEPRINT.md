## Service Pipeline (Source -> Product)
1. Source intake
- Local code/docs/runs (`training/runs/local_cloud_sync/*`, repo docs, scripts).
- Notion technical + marketing pages.
- Dropbox repo mirrors and backups.

2. M5 normalization
- Normalize into event records with source URL/path, summary, risk, six-tongue tags, and mesh node labels.
- Output JSONL for governance and publish pipelines.

3. Governance membrane
- Run SCBE checks and tuned thresholds.
- Keep deterministic artifacts: `audit.json`, `statevector.json`, `decision_record.json`, `summary.json`.

4. Delivery sinks
- Airtable: operational tracking (project, tasks, releases, revenue impact).
- Hugging Face: versioned dataset drops for M5 mesh training.

5. Optional trust layer
- Write content hash + decision signature to decentralized ledger for immutable proof of provenance.

## Six-Tongue Mesh Mapping
- `KO`: intent/control events and orchestrator commands.
- `AV`: transport metadata and source routing.
- `RU`: policy constraints and governance outcomes.
- `CA`: compute features and model-facing tensors.
- `UM`: redaction/security labels and quarantine paths.
- `DR`: schema/version contracts and attestation envelopes.
