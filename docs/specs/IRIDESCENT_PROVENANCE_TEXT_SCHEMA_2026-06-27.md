# Iridescent Provenance Text Schema

Date: 2026-06-27
Status: draft v1
Use cases: AI-assisted book writing, Bible-style colored text, training data curation, research confidence tagging, code/project assignments, SCBE conlang/coding-system overlays

## Purpose

Create a text system where authorship, source truth, research confidence, emotion, code role, and SCBE language/coding context are visible in color and machine-readable in metadata.

The key idea:

```text
word meaning comes from text
word trust comes from provenance
word role comes from context tags
word training value comes from source/confidence labels
word visual identity comes from color layers
```

## Design principle

Do not bake meaning only into color.

Store meaning as structured tags first:

```json
{
  "source": ["user_original"],
  "confidence": ["verified_human"],
  "mode": ["idea"],
  "runtime": ["draft"]
}
```

Then render those tags as:

- Bible-style colored lettering
- iridescent glyph gradients
- underline/border markings
- training metadata
- verifier-readable JSON

## Core layers

### Layer 1 - authorship/source

| Tag | Meaning | Base color |
|---|---|---|
| `user_original` | Words typed directly by Issac. | deep blue |
| `user_voice_guide` | Explicit guidance about Issac's preferred voice/style. | cyan |
| `saved_user_text` | Previously saved user-written material. | royal blue |
| `user_idea` | User-originated idea, sketch, invention, or concept. | teal |
| `ai_output` | Generated by AI. | violet |
| `ai_edited_user` | AI edited but user-originated. | blue-violet blend |
| `human_external` | Human-authored external source. | green |
| `system_generated` | Tool/system-generated logs, receipts, metrics. | gray |

### Layer 2 - evidence/confidence

| Tag | Meaning | Base color |
|---|---|---|
| `verified_human` | Human source verified or directly cited. | green |

| `established_unproven` | Commonly used or accepted but not proven here. | muted yellow |
| `hypothesis` | Working theory. | orange |
| `fictional_or_symbolic` | Mythic, metaphorical, narrative, or symbolic layer. | magenta |

## Word/span model

Default storage should be span-based:

```json
{
  "text": "Use Colab as compute, not truth.",
  "spans": [
    {
      "start": 0,
      "end": 9,
      "tags": {
        "source": ["ai_output"],
        "confidence": ["verified_ai"],
        "function": ["instruction"],
        "project": ["colab"]
      }
    }
  ]
}
```

For special visual/textual importance, use glyph-level overrides:

```json
{
  "word": "truth",
  "glyphs": [
    {"i": 0, "char": "t", "tags": {"confidence": ["established_proven"]}},
    {"i": 1, "char": "r", "tags": {"source": ["user_idea"]}},
    {"i": 2, "char": "u", "tags": {"function": ["constraint"]}},
    {"i": 3, "char": "t", "tags": {"project": ["production"]}},
    {"i": 4, "char": "h", "tags": {"confidence": ["verified_human"]}}
  ]
}
```

## Iridescent rendering

When a word has multiple tag layers, render it as a multi-band gradient:

```text
top band: authorship
middle band: confidence/evidence
bottom band: project/runtime role
underline: citation/receipt state
glow: cutting-edge or symbolic layer
```

When using per-letter striping:

```text
letter 1 = source/authorship
letter 2 = confidence
letter 3 = function
letter 4 = project/runtime
letter 5 = conlang/coding lane
repeat pattern for longer words
```

This lets a single word visually encode phrase/sentence context without losing the actual text.

## Bible-style book rendering

Book pages can use:

- black text for neutral narrative
- red/orange for direct instruction
- blue/cyan for Issac-originated voice/guidance
- violet for AI-generated passages
- green/gold for verified/cited claims
- amber for unverified/hypothesis
- rose for feeling/intuition
- iridescent words for multi-tagged concepts

Important: every colored passage must still have machine-readable tags.

## Training implications

The model can learn:

- distinguish user-originated ideas from AI output
- preserve Issac's voice guides
- preserve raw human spelling/noise while learning likely corrections
- avoid treating unverified research as fact
- separate feelings from claims
- mark cutting-edge discoveries as provisional
- stop before official release if only draft/provisional tags exist
- use SCBE conlang/code lanes as structured semantic context

Correction examples should keep both forms:

```text
coool(cool)
wiht(with)
aethredesk(AetherDesk?)
```

The raw surface form teaches real human distribution. The bracketed form teaches the likely canonical reading.

## Minimum packet

```json
{
  "schema": "scbe.iridescent_text.v1",
  "id": "segment_001",
  "plain_text": "Colab is compute, not truth.",
  "segments": [
    {
      "start": 0,
      "end": 26,
      "tags": {
        "source": ["ai_output"],
        "confidence": ["verified_ai"],
        "function": ["instruction"],
        "project": ["colab", "tool"],
        "runtime": ["draft"]
      },
      "render": {
        "mode": "iridescent_span",
        "palette": ["violet", "emerald", "orange", "cyan", "gray-blue"]
      }
    }
  ],
  "metadata": {
    "created_by": "aetherdesk",
    "provenance": "training_spec",
    "validated": false
  }
}
```

## Storage rules

- Store plain text separately from visual tags.
- Store offsets against plain UTF-8 text.
- Keep original user text immutable when marked `user_original`.
- Allow AI edits only as new segments, not silent overwrites.
- Keep raw traces temporary; curated tagged text is the durable artifact.
- Do not rely on color alone for accessibility or training.
