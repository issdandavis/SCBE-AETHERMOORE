# Video Processing Model Spec

## Purpose

Train and deploy a multi-modal video understanding model that extracts structured intelligence from video content using subtitle-aligned frame sampling, audio transcript alignment, and Sacred Tongue classification. The model turns raw video into governed, tongue-classified training data at 99% less compute than naive frame-by-frame approaches.

Target artifacts:
- **Model**: `issdandavis/scbe-video-understanding-v1`
- **Dataset**: `issdandavis/scbe-video-training-pairs`
- **Space**: `issdandavis/SCBE-Video-Demo`

## 1. Model Architecture

### Inputs (per segment)

| Input | Format | Source |
|-------|--------|--------|
| Video frames | RGB tensors, sampled at subtitle-change boundaries | ffmpeg + subtitle parser |
| Audio transcript | Timestamped text (word-level) | Whisper output |
| Subtitle track | SRT/VTT parsed as temporal index | Embedded or sidecar file |
| Previous context | Last N segment summaries + tongue activations | Model's own prior output |

A "segment" is defined as the interval between two consecutive subtitle text changes. One segment may contain one or more subtitle entries if the text remains unchanged, but a new segment always starts when the displayed text differs from the previous entry.

### Outputs (per segment)

| Output | Type | Description |
|--------|------|-------------|
| Scene description | Free text (1-3 sentences) | What is visually happening in the frame(s) |
| Action classification | Enum + confidence | What activity is occurring (talking, demonstrating, coding, transitioning, etc.) |
| Emotional tone | 6D vector (joy, anger, sadness, surprise, fear, neutral) | Derived from facial expressions + audio prosody |
| Tongue classification | 6D activation vector [KO, AV, RU, CA, UM, DR] | Which Sacred Tongue channels activate for this content |
| Summary | Single sentence | One-line description of the segment's meaning |

### Backbone Components

```
                   +------------------+
                   |  Frame Encoder   |  (CLIP ViT-L/14 or SigLIP)
                   +--------+---------+
                            |
  +-------------+    +------v------+    +----------------+
  | Whisper     |--->| Cross-Modal |<---| Subtitle       |
  | Transcript  |    | Fusion      |    | Temporal Index |
  +-------------+    +------+------+    +----------------+
                            |
                   +--------v---------+
                   | Context Window   |  (previous N segment embeddings)
                   +--------+---------+
                            |
              +-------------+-------------+
              |             |             |
     +--------v---+  +-----v------+  +---v-----------+
     | Scene Head |  | Tone Head  |  | Tongue Head   |
     | (text gen) |  | (6D reg)   |  | (6D classify) |
     +------------+  +------------+  +---------------+
```

The fusion module is a lightweight transformer decoder (6 layers, 512 hidden) that attends over frame embeddings, transcript token embeddings, and subtitle boundary markers. The context window feeds the last 8 segment outputs as prefix tokens, giving the model temporal continuity without full-video attention.

## 3. The Subtitle-Aligned Sampling Insight

### The Problem

Standard video processing samples every frame or at a fixed interval (e.g., 1 fps). For a 10-minute video at 30 fps, that is 18,000 frames. Even at 1 fps, it is 600 frames. Most of these frames are visually redundant -- the speaker's mouth moves but the scene does not change in meaning.

### The Solution

Sample frames only when the subtitle text changes. A subtitle change signals a meaning change -- the speaker said something new, the topic shifted, or the scene transitioned. Between subtitle changes, the visual content is overwhelmingly static or repetitive.

### The Math

| Metric | Naive (30 fps) | Fixed (1 fps) | Subtitle-aligned |
|--------|----------------|---------------|------------------|
| Frames per minute | 1,800 | 60 | 6-15 |
| 10-min video | 18,000 frames | 600 frames | 60-150 frames |
| Compute reduction vs naive | -- | 97% | **99.2-99.7%** |
| Information loss vs 1fps | -- | -- | **Minimal** (meaning-preserving) |

Why minimal information loss:
- Subtitle boundaries already encode semantic transitions
- Visual scenes persist across many frames between subtitle changes
- The rare case where a visual change occurs without a subtitle change (e.g., a scene cut during a pause) can be caught by a simple frame-diff threshold as a fallback detector

### Implementation

```python
def subtitle_aligned_sample(video_path: str, subtitle_path: str) -> list[dict]:
    """Extract frames only at subtitle change boundaries."""
    subs = parse_subtitles(subtitle_path)  # SRT/VTT -> list of (start, end, text)
    frames = []
    prev_text = ""
    for sub in subs:
        if sub.text.strip() != prev_text.strip():
            # Meaning changed -- sample the frame at this timestamp
            frame = extract_frame(video_path, sub.start)
            frames.append({
                "timestamp": sub.start,
                "frame": frame,
                "text": sub.text,
                "prev_text": prev_text,
            })
            prev_text = sub.text.strip()
    return frames
```

Fallback: for videos without subtitles, fall back to 1 fps with frame-diff filtering (skip frames where SSIM > 0.95 compared to previous sampled frame).

## 4. The Dye Injection Layer

### Concept

Every video segment gets tongue-classified. The tongue activation vector is a 6D signal [KO, AV, RU, CA, UM, DR] that characterizes the type of content in that segment. This creates training data for the Sacred Tongue classifier from video, not just text -- extending the tongue model into the visual+audio domain.

### Tongue-Content Mapping

| Content Type | Primary Tongue | Secondary | Signal Pattern |
|--------------|---------------|-----------|----------------|
| Technical explanation (code, math, architecture) | **CA** (Compute) | AV (Metadata) | CA spikes to 0.7-0.9 |
| Story/narrative (plot, characters, worldbuilding) | **RU** (Binding) | DR (Structure) | RU spikes to 0.6-0.8 |
| Security/threat discussion (attacks, defenses, CVEs) | **UM** (Security) | CA (Compute) | UM spikes to 0.7-0.9 |
| Instructions/commands (tutorials, how-to, setup) | **KO** (Intent) | CA (Compute) | KO spikes to 0.6-0.8 |
| Metadata/context (intros, credits, channel plugs) | **AV** (Metadata) | KO (Intent) | AV spikes to 0.5-0.7 |
| Structural transitions (chapter breaks, recaps) | **DR** (Structure) | AV (Metadata) | DR spikes to 0.5-0.7 |

### Why This Matters

1. **Cross-modal tongue training**: Currently the tongue classifier only works on text. Video-derived tongue labels extend coverage to visual and audio modalities.
2. **Content fingerprinting**: A video's tongue activation timeline becomes a semantic fingerprint. Two videos with similar tongue timelines cover similar content types in similar order, even if the words differ.
3. **Governance integration**: Tongue activations feed directly into the L13 risk decision tier. A video segment with UM spike followed by KO spike (security threat then instruction) gets flagged for review -- someone might be teaching an exploit.
4. **Training data multiplication**: Every video processed generates tongue-labeled training pairs. The more video the model processes, the better the tongue classifier becomes.

### Dye Injection Pipeline

```
Video Segment
    |
    +---> Transcript text ---> Sacred Tongue Tokenizer ---> Text activation [6D]
    |
    +---> Frame features ---> Tongue Vision Head ---> Visual activation [6D]
    |
    +---> Audio prosody ---> Tongue Audio Head ---> Audio activation [6D]
    |
    +---> Weighted fusion (text=0.5, visual=0.3, audio=0.2) ---> Final tongue vector [6D]
    |
    +---> Governance check (L13 risk tier)
    |
    +---> Store as training pair for next iteration
```

The fusion weights (0.5/0.3/0.2) are initial heuristics. Phase 3 training will learn optimal weights from labeled data.

## 5. Training Plan

### Phase 1: Frame Captioning Model

**Goal**: Given a single video frame, produce an accurate scene description.

| Aspect | Detail |
|--------|--------|
| Base model | LLaVA-1.6-Mistral-7B (or LLaVA-NeXT for smaller footprint) |
| Fine-tune method | LoRA (rank 16, alpha 32) on the projection + language layers |
| Training data | ~5,000 (frame, description) pairs from Tier 1+3 sources |
| Epochs | 3 |
| Hardware | Colab T4 (16GB VRAM) -- fits with LoRA + 4-bit quantization |
| Output | `issdandavis/scbe-frame-captioner-v1` |
| Eval metric | BLEU-4 + human spot-check on 100 held-out frames |
| Timeline | 2-3 weeks (data prep + training + eval) |

### Phase 2: Temporal Alignment Model

**Goal**: Given a sequence of frames + transcript + subtitle index, produce segment summaries, action labels, and emotional tone.

| Aspect | Detail |
|--------|--------|
| Architecture | Custom: CLIP encoder (frozen) + 6-layer transformer decoder with cross-attention over transcript tokens |
| Training data | ~15,000 (frame sequence, transcript, summary, action, tone) tuples |
| Key innovation | Subtitle boundaries as explicit position tokens in the transformer sequence |
| Hardware | A100 40GB (Colab Pro or Lambda Labs, ~$1.10/hr) |
| Training time | ~8 hours on A100 |
| Output | `issdandavis/scbe-temporal-align-v1` |
| Eval metric | ROUGE-L for summaries + accuracy for action classification + MSE for tone vectors |
| Timeline | 4-6 weeks (after Phase 1 provides frame descriptions as features) |

### Phase 3: Tongue Classification on Video

**Goal**: Given frames + audio, predict the 6D Sacred Tongue activation vector.

| Aspect | Detail |
|--------|--------|
| Architecture | Multi-head classifier on top of Phase 2 embeddings + audio features (wav2vec2) |
| Training data | ~20,000 segments with tongue labels (from Dye Injection pipeline running on Phase 1+2 outputs) |
| Key innovation | First model that classifies Sacred Tongue activations from video, not text |
| Hardware | A100 40GB |
| Training time | ~4 hours on A100 |
| Output | `issdandavis/scbe-video-tongue-v1` |
| Eval metric | Cosine similarity between predicted and ground-truth tongue vectors (target > 0.85) |
| Timeline | 3-4 weeks (after Phase 2 provides temporal embeddings) |

### Combined Model

After Phases 1-3, merge the three into a single inference pipeline:

```
issdandavis/scbe-video-understanding-v1
    = Phase 1 captioner (LoRA weights)
    + Phase 2 temporal aligner (full weights)
    + Phase 3 tongue classifier (head weights)
```

The combined model runs in a single forward pass per segment. Input: frame(s) + transcript + subtitle boundaries. Output: all five target fields (scene description, action, tone, tongue activation, summary).

## 8. Integration with SCBE Stack

### Governance Pipeline

Video segments flow through the standard 14-layer pipeline:
- L3-4: Transcript text gets tongue-weighted embedding
- L5: Hyperbolic distance from safe-operation centroid
- L12: Harmonic wall score `H(d, pd) = 1/(1+d_H+2*pd)` using tongue activation as the polyhedral dimension
- L13: Risk decision (ALLOW/QUARANTINE/ESCALATE/DENY) on each segment

A video with DENY segments gets those segments redacted in the output. QUARANTINE segments are flagged for human review.

### Training Data Flywheel

```
Videos in  --->  Subtitle-aligned sampling  --->  Model inference
                                                       |
                                                       v
                                               Tongue-labeled segments
                                                       |
                                                       v
                                               Governance check (L13)
                                                       |
                                               +-------+-------+
                                               |               |
                                           ALLOW           QUARANTINE/DENY
                                               |               |
                                               v               v
                                      Add to training set   Flag for review
                                               |
                                               v
                                      Push to HF dataset
                                               |
                                               v
                                      Retrain model (next version)
```

Every video processed makes the model better. The governance gate ensures that adversarial or unsafe content does not contaminate the training set.

## 11. Open Questions

1. **Subtitle availability**: Not all videos have subtitles. The frame-diff fallback works but produces noisier boundaries. Should we train a separate boundary-detection model?
2. **Tongue label bootstrapping**: Phase 3 depends on tongue labels generated by the text tokenizer. If the text tokenizer has biases, they propagate. Need a human-labeled validation set for tongue activations.
3. **Multi-speaker segments**: Current design treats each segment as single-speaker. For panel discussions or interviews, need speaker diarization (pyannote-audio) as a preprocessing step.
4. **Copyright**: Tier 2 data uses YouTube auto-captions and extracted frames under fair use for research/training. Need legal review before any commercial deployment of models trained on external content.
5. **Tongue weight learning**: The fusion weights (text=0.5, visual=0.3, audio=0.2) are heuristic. Should Phase 3 learn these jointly, or should they be fixed as a governance constraint?
