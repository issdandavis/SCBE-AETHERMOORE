# OpenClaw Bootstrap

Use this to initialize OpenClaw quickly without wasting turns on generic onboarding.

## Identity

- Call the user `Issac`.
- Timezone: `America/Los_Angeles`.
- Keep tone direct and practical.

## Project

`SCBE-AETHERMOORE` is a large multi-system repo for governed AI orchestration, HYDRA lanes, Sacred Tongues and tokenizer work, training-data pipelines, research automation, browser and agent tooling, and monetization experiments.

## Working Rules

- Build first, document after.
- Turn speculation into hypotheses, tests, and results.
- Prefer repo truth over abstract summaries.
- Do not spend time on generic rapport-building once identity and project are known.
- DARPA work, docs, and training data may live in the repo intentionally. Do not assume they are accidental clutter.
- Local git hygiene is handled separately from repo policy.

## Current Priorities

- Improve HYDRA and OpenClaw lanes.
- Build better experiment and test loops.
- Keep local git hygiene usable without deleting intentional work.
- Favor deterministic orchestration over vague agent behavior.

## Context Retrieval Rule

If more context is needed:

1. Read the repo and summarize actual live entrypoints first.
2. Prefer code, scripts, and tests over theory notes.
3. Separate:
   - what exists now,
   - what is hypothesized,
   - what is already measured.
