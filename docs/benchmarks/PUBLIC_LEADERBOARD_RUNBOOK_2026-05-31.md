# Public Leaderboard Runbook

Generated: 2026-05-31

This is the practical path for turning SCBE benchmark work into public results that technical reviewers can inspect without laughing at the claim boundary.

## Rule

Do not call a local fixture a public benchmark result.

A public claim needs all of the following:

- official benchmark harness or an unchanged upstream fork;
- exact SCBE commit hash;
- exact agent package version and model/provider version;
- command line used to run the benchmark;
- raw per-task output artifacts;
- pass/fail summary generated by the benchmark harness;
- notes on blocked tasks, skipped tasks, retries, and environment failures.

## First Public Targets

| Priority | Suite | Why It Fits SCBE | Public Claim If We Pass |
| --- | --- | --- | --- |
| 1 | Terminal-Bench 2.x | terminal agents, Docker tasks, task-level pass/fail; closest to the 13/13 local parity evidence | SCBE governed terminal agent completes N/M official tasks with command-level governance enabled |
| 2 | AgentDojo | tool-use prompt-injection benchmark with attacks and defenses | SCBE prevents unsafe tool actions under indirect prompt-injection conditions |
| 3 | tau-bench / tau2-bench | multi-turn tool, user, policy, and database-state evaluation | SCBE preserves policy compliance and consistency across customer-service style tasks |
| 4 | WorkArena / BrowserGym | enterprise browser workflows and action traces | SCBE governs browser actions across long-horizon enterprise UI tasks |
| 5 | OSWorld | desktop and app-control benchmark | SCBE governs real computer-control actions, not only shell commands |

## AgentDojo Public Run

AgentDojo is the best governance/security target because it evaluates prompt injection attacks and defenses for tool-using agents.

Baseline install and smoke:

```powershell
python -m venv .venv-agentdojo
.venv-agentdojo\Scripts\python -m pip install --upgrade pip
.venv-agentdojo\Scripts\python -m pip install agentdojo
.venv-agentdojo\Scripts\python -m agentdojo.scripts.benchmark --help
```

Adapter work:

- implement a SCBE defense pipeline that checks every tool call and tool result;
- map ALLOW, QUARANTINE, ESCALATE, DENY to AgentDojo defense behavior;
- persist every decision as a governance event;
- run one small suite first, then the full benchmark.

Example upstream command shape:

```powershell
python -m agentdojo.scripts.benchmark -s workspace -ut user_task_0 -ut user_task_1 --model gpt-4o-2024-05-13 --defense tool_filter --attack tool_knowledge
```

SCBE target:

```powershell
python -m agentdojo.scripts.benchmark -s workspace --model <model> --defense scbe_govern --attack tool_knowledge
```

## tau-bench / tau2-bench Public Run

This is the best product benchmark for policy-following agents. It tests whether an agent can finish multi-turn customer-service tasks while following domain-specific policy and updating the database correctly.

Adapter work:

- wrap tool calls with `scbe-govern`;
- add policy-decision metadata per call;
- preserve final database-state scoring from the upstream harness;
- report both completion and policy/governance metrics.

SCBE-specific metrics to add beside the official score:

- task success rate;
- pass^k or retry reliability when available;
- policy violation count;
- unsafe tool-call block count;
- false block count on required benign tool calls;
- average turns and total tool calls;
- governance event completeness.

## Browser/Desktop Public Runs

WorkArena, BrowserGym, and OSWorld matter for long-form "agent movement" because they evaluate real browser or desktop actions instead of clean text-only tool calls.

SCBE adapter target:

- before each click, type, navigation, file write, or API call: score the action;
- DENY irreversible or cross-boundary actions;
- QUARANTINE high-risk actions while preserving the upstream action trace;
- attach a governance event ID to each action in the run log.

Start with WorkArena/BrowserGym before OSWorld. Browser tasks are easier to reproduce and publish than full desktop-control tasks.

## Publish Packet

For every public run, publish a small static packet:

```text
docs/benchmarks/public-runs/<suite>/<date>/
  README.md
  run_command.txt
  environment.json
  results.json
  governance_summary.json
  per_task/
```

The website should link to the packet and state only the benchmark version, score, commit, and claim boundary.
