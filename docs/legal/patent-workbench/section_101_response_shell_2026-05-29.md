# Section 101 Response Shell - SCBE-2026-0001

Status: prosecution-prep draft, not legal advice, not filed.

Application: 19/691,526  
Docket: SCBE-2026-0001  
Use: starting point if an Office Action asserts that claim 1 is directed to an abstract idea or mathematical concept.

## Core Position

Claim 1 is not directed to a mathematical result in isolation. It is directed to a specific computer-implemented runtime execution-control technique for AI and agentic systems. The claimed process uses bounded geometric state, persisted session state, nonlinear governance cost, and execution routing to change whether a proposed computational action is allowed, reviewed, quarantined, rerouted, or denied.

The claim should be framed as:

> a specific technique for controlling execution of computational actions in AI/agentic systems using Poincare-ball embedding, session-centroid drift measurement, and a governance decision gate.

Do not frame the claim as:

> safe AI, alignment proof, ownership of hyperbolic geometry, or a mathematical discovery.

## Alice Step 1 - Practical Application

The claim integrates mathematical calculations into a practical application because the calculations are used to control a computer-execution path.

Technical features to emphasize:

1. The proposed action is encoded into structured machine state.
2. The action state is embedded or scored in a bounded geometric domain.
3. The system compares the action against persisted session/reference state.
4. The system computes nonlinear governance cost and auxiliary signals.
5. The system routes execution to allow, review, quarantine, reroute, or deny.
6. The system records audit/tamper/receipt outputs or fail-to-noise denial output.

This is the SRI/Enfish/McRO posture:

- specific technique;
- specific computer-security/runtime problem;
- specific machine behavior change;
- not a result-only claim.

## SRI Mapping

SRI International v. Cisco supports the Step 1 frame where claims use a specific technique to solve a technological problem in computer networks.

| SRI pattern | SCBE pattern |
|---|---|
| multiple monitors analyze network data | pre-filters, geometry gate, tamper/canonicality overlays analyze proposed computational action |
| reports are integrated | drift, cost, trust, tamper, and policy signals are aggregated |
| network intrusion problem | AI/agentic execution drift and tool-misuse problem |
| output identifies intruders | output routes execution to allow/review/quarantine/reroute/deny |

Prosecution sentence:

> Like the network-security claims in SRI, claim 1 recites a specific technique for solving a technological problem. The technique is not merely classifying information; it controls whether and how a proposed computational action executes.

## Enfish Mapping

Enfish supports the argument that a computer-related claim is not abstract when its focus is an improvement to computer functionality itself.

SCBE improvement:

- A conventional guardrail can inspect text or block strings.
- SCBE improves runtime governance by maintaining a geometric session reference and changing execution based on measured drift from that reference.
- The improvement is to the operation of the AI/agentic runtime control layer, not to a human business method performed on a computer.

## McRO Mapping

McRO supports specific mathematical rules when they are a particular means for achieving a technical improvement.

SCBE mathematical rules:

- Poincare-ball distance or bounded geometric drift;
- session centroid update;
- nonlinear governance cost;
- phi/axis weighting in dependent embodiments.

Response caution:

- Do not say formulas alone are patentable.
- Say the formulas are the specific means by which the runtime control system measures drift and routes execution.

## Alice Step 2 Fallback - Ordered Combination

If the examiner says the claim is abstract at Step 1, the BASCOM fallback is that the ordered combination is non-conventional and non-generic even if some components were known individually.

Ordered combination to preserve:

1. cheap pre-filter stack;
2. structured action encoding;
3. bounded geometric embedding/scoring;
4. session-centroid/reference comparison;
5. nonlinear cost;
6. tamper/canonicality overlays;
7. execution routing;
8. audit/receipt/fail-to-noise output;
9. durable state continuity.

Prosecution sentence:

> The inventive concept is not any one known component, but the ordered arrangement that uses persisted hyperbolic runtime state to govern execution of proposed AI/agentic actions.

## Berkheimer Evidence Hook

If the examiner asserts the limitations are well-understood, routine, and conventional, request evidentiary support for that factual assertion and point to SCBE evidence showing the ordered combination has measured technical effects.

Evidence to attach or cite when ready:

- benchmark table comparing raw model, regex-only, KO/tongue coverage gate, RuntimeGate overlays, and full route;
- persistence tests showing session state survives restart;
- tamper/canonicality tests showing concrete computer-security behavior;
- quarantine/reroute tests showing execution behavior changes.

## Claim Language To Protect

Keep central:

- proposed computational action;
- bounded geometric domain;
- session/reference state;
- nonlinear governance cost;
- execution decision;
- quarantine/reroute/deny;
- audit/tamper/receipt output.

Avoid making these central:

- "safe AI";
- "alignment";
- "consciousness";
- "sacred" labels;
- "unhackable";
- "guarantee."

## Filing Response Shape

If a 101 rejection arrives, a response can be structured as:

1. Claim 1 is not directed to an abstract idea because it recites a specific runtime execution-control technique.
2. The technique is integrated into a practical application by controlling execution of proposed computational actions.
3. Even if the claim were found abstract, the ordered combination provides an inventive concept under BASCOM.
4. Any assertion that the ordered combination is well-understood, routine, and conventional requires evidentiary support under Berkheimer.

Sources to verify before filing a response:
