# Patent Global Scope Research - SCBE Root Filing

Status: research notes, not legal advice.

Application anchor: `63/961,403 | SCBE-2026-0001`

Title: `System and Method for Hyperbolic Geometry-Based Authorization with Topological Control-Flow Integrity`

## Question

Can SCBE be handled as one broad patent filing that preserves the system as a
whole while leaving room for later continuation, divisional, PCT, South Korea,
China, and other national filings?

## Short Answer

The best structure is not literally "one patent to rule them all." It is:

1. one strong root specification that discloses the shared invention spine;
2. one initial claim set aimed at the strongest common inventive concept;
3. continuation/divisional filings if the examiner restricts different invention
   groups;
4. optional PCT by the 12-month priority deadline if international rights matter;
5. national phase entries later only where the cost/business case is real.

The root application can be the master disclosure. The claims should still be
kept focused enough to reduce restriction/unity risk.

## Official Sources

### United States / USPTO

- USPTO and MPEP materials describe the PCT as a path for seeking protection in
  many countries through one international application, generally filed within
  12 months of the first application when claiming priority.
- USPTO guidance for foreign filings says an invention made in the United States
  generally needs a foreign filing license before filing abroad, unless the
  applicant waits six months after the U.S. filing and no secrecy order applies,
  or the USPTO has granted the license earlier.

Practical effect for SCBE:

- The U.S. non-provisional conversion deadline remains January 15, 2027.
- If preserving PCT / foreign options from the January 15, 2026 provisional
  matters, treat January 15, 2027 as the PCT decision deadline too.
- Before any foreign or international filing outside the USPTO receiving office,
  check the foreign filing license status on the USPTO filing receipt.

### PCT / WIPO

WIPO states that a PCT application can be filed within 12 months from the
earliest application for the same invention to claim priority. WIPO also frames
PCT as a single international application path that delays national phase cost
and decisions, not as one worldwide patent.

Practical effect for SCBE:

- PCT is the "keep options open" tool.
- It does not itself grant a patent.
- It buys time before national phase decisions, commonly around 30 months from
  priority for many jurisdictions, subject to each national/regional office.

### South Korea / KIPO

KIPO's English filing procedure says a patent application claiming priority
should be filed in Korea within one year from the priority application. KIPO
also lists the normal application parts: application bibliographic data,
specification, drawings if any, abstract, priority document/translation if
requested, and power of attorney if necessary.

Practical effect for SCBE:

- Korea should be treated as a translation/local-formality jurisdiction.
- If using PCT, later Korean national phase planning should be checked against
  WIPO/KIPO current requirements before spending money.

### China / CNIPA

CNIPA's English Patent Law translation states that an invention or utility model
application filed in China can claim foreign priority within twelve months for
the same subject matter. CNIPA materials also require priority declaration and
priority document compliance. CNIPA PCT/national phase materials should be
checked before any China phase decision.

Practical effect for SCBE:

- China requires careful translation and local rules.
- "Same subject matter" and original disclosure support matter; later-added
  features may not get the January 15, 2026 priority date.
- China planning should be separated into a cost/business decision, not assumed.

## Reddit / Practitioner Signal

Reddit is not authority, but law/patent communities repeatedly surface useful
failure patterns:

- Trying to claim too many distinct inventions in one application often triggers
  restriction or lack-of-unity problems.
- A broad root specification can disclose multiple embodiments, but the first
  claim set should usually focus on one common inventive concept.
- Examiners/practitioners emphasize support under section 112: high-level
  software flowcharts without enough "how" can be attacked.
- PCT is often described by practitioners as a time-buying system, not a cheap
  worldwide patent.

Practical effect for SCBE:

- Put the whole system in the specification where supported.
- Keep the first independent claims tied to the same technical spine:
  hyperbolic authorization + topological/control-flow integrity + governed
  runtime decisioning.
- Put newer or side inventions into dependent claims, examples, or future
  continuation/CIP buckets depending on support.

## SCBE Root Filing Strategy

### Root Spine

The common inventive concept should be:

> A computer-implemented governance system that maps an action/context into a
> non-Euclidean authorization state, measures drift or distance from trusted
> operation, applies nonlinear/harmonic cost scaling and topological integrity
> checks, and gates execution through allow/review/quarantine/deny decisions.

This is the "one field" that keeps the application coherent.

### Claim Families That Fit The Root

- hyperbolic governance gate;
- harmonic cost scaling;
- semantic weighting axes;
- topological/control-flow integrity;
- runtime decision gate;
- audit receipt;
- quarantine containment;
- bijective tamper / canonical identifier signals, if priority support exists.

### Likely Continuation / Divisional Buckets

These may belong in the disclosure, but should be watched for restriction or
new-matter risk:

- agent-bus orchestration and workflow fleet systems;
- rotating drop-box key authorization;
- resonant thought lattice / reasoning effort rings;
- Sacred Tongues tokenizer/transport as a separate compiler or encoding system;
- audio/vision/synesthetic overlays;
- government/fleet operational surfaces.

## "One Patent To Rule Them All" Translation

Use this internal phrase as:

> One root disclosure to preserve the system architecture, with claim families
> routed through continuation/divisional/PCT strategy as prosecution requires.

Do not use it as:

> One issued patent will legally cover every SCBE subsystem everywhere.
