## Current Drag

The main repo currently loses trust because:

- Too many GitHub workflows overlap.
- Generated artifacts and experiments sit close to canonical code.
- Multiple docs describe similar ideas at different maturity levels.
- Some deployment material is real, but not yet presented as one clean path.
- The public landing path does not yet force readers through the strongest evidence first.

## Compression Rules

Do not delete useful work. Classify it.

Every main-repo file should fit one of these categories:

1. Functional code: imported, executed, tested, or used by a script/workflow.
2. True documentation: explains current behavior, operating instructions, proposal support, or known roadmap.
3. Deployment configuration: manifests, workflows, Docker, Kubernetes, CI, Pages, or cloud helpers.
4. Training/evaluation asset: dataset manifest, eval harness, scoring tool, or adapter registry.
5. Generated artifact: ignored or clearly downstream of a reproducible script.
6. Research/archive: preserved, but not allowed to define the current system.

## Immediate GitHub Cleanup Passes

### Pass 1: Workflow Consolidation

Goal: reduce noisy automation while keeping cheap/free gates.

Authority targets:

- `ci.yml` for build/test.
- One CodeQL/security authority.
- One Pages deployment authority.
- One PR governance authority.
- One nightly/weekly maintenance authority.
- One manual cloud-operator authority for free or already-paid lanes.

### Pass 2: Public Evidence Path

Goal: make the strongest external path obvious.

Add or strengthen:

- `docs/rfi-support/SCBE_AETHERMOORE_Execution_Environment_Appendix_20260426.md`
- `docs/rfi-support/GITHUB_DEPLOYMENT_WORKFLOW_AUDIT_2026-04-26.md`
- `docs/rfi-support/SALES_TREAT_PACKET_INDEX_20260426.md`
- A short public "proof path" page that links demos, benchmarks, deployment surface, and training/eval status.

### Pass 3: Generated and Research Boundary

Goal: stop generated outputs from looking like canonical source.

Actions:

- Keep `artifacts/` ignored.
- Keep local sales packets under `artifacts/contracts/`.
- Mirror only indexes or summaries into tracked `docs/rfi-support/`.
- Move obsolete "master" docs into archive pointers instead of letting them compete with current docs.
