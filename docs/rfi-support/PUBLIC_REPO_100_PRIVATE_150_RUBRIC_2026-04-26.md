# Public Repo 100 / Private System 150 Rubric

Date: 2026-04-26

## Purpose

SCBE-AETHERMOORE needs two quality bars:

- Public GitHub repo: 100/100 proof surface.
- Private military-grade lanes: 150/100 internal capability, with deeper action sequences, coding regimens, and controlled DARPA-facing material.

The public repo must be clean enough that an outside reviewer trusts the system before seeing private work.

## Public Repo 100/100 Standard

The public repository reaches 100 only when every visible surface fits one of these categories:

1. Functional code: imported, executed, tested, packaged, or used by a documented entrypoint.
2. True documentation: describes current behavior, instructions, deployment, guidance, roadmap, or known limitations.
3. Deployment configuration: GitHub Actions, Kubernetes, Docker, Pages, or free/included cloud helpers with clear operating boundaries.
4. Training/evaluation asset: dataset manifest, scoring tool, benchmark, adapter registry, or promotion gate.
5. Generated artifact pointer: tracked only as an index or summary; heavy generated outputs stay ignored.
6. Research/archive pointer: preserved as context, but not allowed to define current system behavior.

## Public Score Gates

| Lane | 100/100 Requirement |
| --- | --- |
| README and public story | One clear start path, one proof path, no unsupported claims. |
| CI | Main branch and PR checks are green or intentionally scoped. |
| Workflows | No duplicate automatic workflows doing the same job. |
| Deployment | GitHub Pages works; Kubernetes manifests render locally; paid cloud is opt-in only. |
| Security | Secret scanning, dependency checks, and CodeQL have one authority path each. |
| Docs | Canonical docs do not conflict with runtime/test reality. |
| Training/eval | Claims point to manifests, reports, or runnable scorers. |
| Generated outputs | Ignored or summarized, not mixed into authority docs. |

## Private 150/100 Standard

Private lanes can exceed public capability by containing:

- Military-grade action sequences.
- Controlled agent fleet procedures.
- Deeper DARPA proposal packets.
- Stress-test scenarios and failure drills.
- Internal code regimens and training recipes.
- Sensitive partner, contract, and outreach context.

Private lanes must not leak into public claims unless the matching public proof exists.

## Claim Discipline

The system can describe what it does, but public claims need evidence:

- "Implemented" requires code or manifests.
- "Tested" requires a test command or report.
- "Deployable" requires a render/build/run path.
- "IL5-ready" is not allowed unless inside an authorized boundary.
- "Military-grade" stays private unless backed by a public standard or contract-approved evidence.

## Immediate Upgrade Path

1. Consolidate duplicate GitHub workflows.
2. Make one public proof path from README to docs/rfi-support.
3. Keep generated artifacts ignored and publish only indexes.
4. Convert duplicate/old docs into pointers or archives.
5. Add public smoke checks for each main deployment surface.
6. Keep private 150/100 systems in gated repos or ignored local artifacts until safe to publish.
