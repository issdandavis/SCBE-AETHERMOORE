KDP Upload Checklist

            

              These are source-linked checkpoints, not legal advice. The app should hyperlink Amazon's current pages so
              the user can verify the rule before upload.
            

          

          

            

              
Manuscript

              
    
                
  • Final trim size chosen before cover is built.
    • 
                
  • Bleed and margins checked for print layout.
    • 
                
  • Images are at least 300 DPI for print quality.
    • 
                
  • eBook manuscript does not include the cover image inside the file.
    • 
              

            

            

              
Cover

              
    
                
  • Paperback/hardcover cover is a single PDF: back + spine + front.
    • 
                
  • Page count is final before cover wrap is generated.
    • 
                
  • No spine text unless page count is over 79 pages.
    • 
                
  • Kindle cover is JPEG/TIFF, ideal 2560 x 1600 px, under 50 MB.
    • 
              

            

            

              
Publishing Form

              
    
                
  • Title, subtitle, author, description, categories, and keywords ready.
    • 
                
  • AI-generated text/images/translations disclosed when required by KDP.
    • 
                
  • AI-assisted editing/brainstorming separated from AI-generated final content.
    • 
                
  • Final preview checked in KDP Print Previewer before publishing.
    • 
              

            

          

        


        

          

            
Cover Lab

            

              KDP's cover tools are the fallback. The better workflow is book-derived: read the manuscript, outline the
              visual scene, generate roughs, choose a direction, then build the print wrap in layers.

Layered Prompt Pack

              

            

          

          

            

              
Wave 1: Roughs

              

                Generate 4-8 small compositions from the book motif. Judge only silhouette, shelf signal, and emotion.
                Do not add title text yet.
              

            

            

              
Wave 2: Layer Build

              

                Generate front art, background extension, spine texture, and back-cover atmosphere separately. Composite
                them into the Bookforge cover wrap after page count is final.
              

            

            

              
Wave 3: Type + KDP Proof

              

                Add title, author, subtitle, spine text, barcode safe area, and blurb in layout software or code. Then
                verify dimensions with the KDP calculator and Print Previewer.
              

            

          

          
    
            
  • 
              User choice matters: the app should let the writer pick the strongest rough before
              spending time on final art.
            
    • 
            
  • 
              Book contents drive the art: the prompt should come from manuscript themes, character
              pressure, setting, motifs, and genre shelf expectations.
            
    • 
            
  • 
              Generated art should have no text: typography belongs in the final cover layout, not in
              the image model output.
