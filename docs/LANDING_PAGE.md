## How It Works Section

### Get Protected in 3 Steps

#### Step 1: Install
```bash
npm install scbe-aethermoore
# or
pip install -r requirements.txt
```

#### Step 2: Seal Your Data
```python
from scbe import SCBE14LayerPipeline

pipeline = SCBE14LayerPipeline()
sealed = pipeline.seal(
    data="sensitive-payload",
    context={"user": "alice", "role": "admin"}
)
```

#### Step 3: Retrieve Securely
```python
result = pipeline.retrieve(sealed, {"user": "alice"})
# Returns: {"decision": "ALLOW", "data": "sensitive-payload"}
```

## FAQ Section

### Frequently Asked Questions

**Q: Is SCBE really quantum-resistant?**
A: Yes. Layer 12 uses ML-KEM-768 (Kyber), a NIST-approved post-quantum algorithm. It's resistant to both Shor's and Grover's algorithms.

**Q: How does context-binding work?**
A: When you seal data, SCBE binds it to a context (user, role, time, location, etc.). On retrieval, the context must match—or access is denied. Stolen credentials are useless.

**Q: What's the performance impact?**
A: Minimal. Full 14-layer seal/retrieve takes <5ms. You can disable layers for even faster performance (not recommended for production).

**Q: Is it HIPAA/SOC 2 compliant?**
A: Yes. SCBE includes built-in compliance features for HIPAA, SOC 2, ISO 27001, and FIPS 140-3. We provide audit documentation.
