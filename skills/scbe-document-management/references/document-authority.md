# Document Authority

Use this order when documents disagree:

1. `docs/specs/CANONICAL_FORMULA_REGISTRY.md`
2. `CANONICAL_SYSTEM_STATE.md`
3. active runtime code and proving tests
4. operational docs
5. public docs
6. research notes
7. historical / handoff docs
8. generated evidence and storage-heavy outputs

Operational interpretation:

- Canonical files define current truth.
- Operational files explain how to run the system.
- Public files explain the system outwardly but cannot override canonical files.
- Research and historical files preserve context but do not own runtime behavior.
